网络概述

计算机网络基本概念

Data

以01序列表示数据 数据不是信息本身
解析数据

Data Packets

为便于理解，给定标准规范传输数据的解析方式
对不同传输层次设定不同的传输数据格式
把数据段分成小的数据报
数据报可走不同传输路径 保证效率

Protocl

5.6.7层对应协议Telnet FTP-文件传输协议 SMTP-简单邮件传输协议 SNMP HTTP
TFTP-简单文件传输协议
4层对应TCP UDP 3层IP
协议定义了在两个或多个通信实体之间交换报文的格式和顺序
以及报文发送或接收报文或其他事件所采取的动作

IP/TCP协议

定义了在路由器和端系统之家发送和接收的分组格式
主要解决通信双方寻址的问题
TCP传输控制协议 HTTP使用TCP作为通信协议 TCP能保证数据不丢失

DNS-域名解析系统

把IP地址转换为便于人类记忆的协议
全称域名系统，是一个将域名和IP地址相互映射的分布式数据库

URL/URI

URI全称统一资源标识符，使用它能够唯一地标记互联网资源
URL全称统一资源定位符，俗称网址，是URI的一个子集

HTTPS

HTTP 明文传输，容易被窃取
HTTPS是以安全为目标的HTTP通道，在HTTP的基础上通过传输加密和身份认证
保证了传输的安全性，HTTPS在HTTP基础上加了SSL层

HTTP 超文本传输协议

网页传输
超文本：图片 音频 文字 视频等 语义被扩大后的文本
文本是一种能被计算机解析的有意义的二进制数据包
网络协议：网络中传递,管理信息的规范
HTTP是在一个计算机世界里专门在两点之间传输超文本数据的约定和规范

组成

1.起始行 描述请求或响应的基本信息
2.头部字段 使用key-value形式更详细地说明报文
3.消息正文 实际传输的数据 它不一定是纯文本 可以是图片 视频等二进制数据
起始行和头部字段合称请求头或响应头统称Header，消息正文饺子实体body
头信息必须有 body可无，在header和body之间必须有一个空行CRLF
HTTP报文起始行由：方法 URL字段和HTTP版本字段构成
####
网络设备 智能手机 电脑等 被称为主机或端系统
端系统之间通过通信链路和分组交换机连接到一起
通信链路指的是因特网中两个节点之间的物理通道 其传输介质有双绞线 光纤 微波
分组交换机：网络交换设备 主要包括交换单元 接口单元 控制单元
路由器和链路层交换机都是分组交换机
l链路的传输速率 一bit/s进行度量

帧frame

数据链路层把网络层交下来的数据构成帧发送到链路上
以及把接收到的帧取出，并上交给网络层

OSI(Open System Interconnection)

一个参考模型 描述数据从发送点如何通过网络到达接收点
同一节点相邻层之间通过接口进行通信

1.Physical 二进制传输
2.Data link access to media
3.Network 路径选择
4.Transport 终端到终端的通讯
5.Session Inter-host connection
6.Presentation
7.Application 提供交互接口
5.6.7层合称应用层其他4层称为数据流层
位于应用层的信息分组称为报文message

1 信号和介质

规范了用于信号传输的介质及传输的标准
主要功能：利用传输介质为数据链路层提供物理连接 实现bit流的透明传输

2 帧和介质访问控制

对传输信号进行检查 对介质访问做管理
封装成帧 透明传输 差错检测
特定链路上数据逻辑控制

3 找最优路径

IP地址是逻辑地址 基于ip地址来进行逻辑定位
实现设备到设备的连接

4 Reliability Flow control Error correction

把数据分成数据段 检查数据是否出错
因特网的运输层在应用程序断点之间传送应用程序报文 第4层报文称为segment

5 dialog and conversations

进程层面的逻辑控制
会话层 建成 管理 终止会话过程

6 common format

数据处理涉及数据压缩数据加密

7 browser

处理用户界面的问题
把用户操作封装成机器所能理解的语法和标准 变成网络通讯内容
上层向下层请求服务

局域网设备

网络终端设备
网络中间设备
主机接入网络依靠网卡NIC(网络适配器)
负责将计算机连接到网络上，可连接光纤，网线
第二层设备 能够识别第二层数据单元帧
帮助主机接入网络 网卡携带一个唯一的mac地址
网卡通过网线和外部网络相连
media线路
第一层设备
中继器 识别信号再把信号放大再转发repeaters
hubs 多端口中继器
第二层设备
网桥Bridges 两个端口
交换机 结合bridges和hub的功能
第三层设备
路由器 网段划分
实现不同网段之间的连接和转换
高层设备能识别低层设备的数据逻辑
